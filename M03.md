## 2. Explicación técnica (visión precisa y comprobable desde el enfoque Cisco)

### 1. Ataques de Capa 2 (Switching)

#### ARP Spoofing / ARP Poisoning

Consiste en enviar respuestas ARP falsas para que una víctima envíe su tráfico al atacante.

Ejemplo aplicado:  
El atacante envía:  
“Mi MAC es la del gateway”  
→ La víctima reenvía todo su tráfico hacia él.  
Resultado: MITM (Man-in-the-Middle).

Es uno de los ataques más preguntados en CCST.

#### MAC Flooding (Ataque a la tabla CAM)

El atacante inunda la tabla CAM del switch con miles de MAC falsas.  
Cuando la tabla se llena, el switch actúa como hub → reenvía tráfico a todos.  
Esto habilita sniffing.

Mitigación: **Port Security**.

#### VLAN Hopping

El atacante intenta saltar de una VLAN a otra mediante técnicas como:

- Switch spoofing
    
- Double-tagging (doble etiquetado 802.1Q)
    

Punto importante del examen:  
Este ataque requiere mala configuración en el switch.

---

### 2. Ataques de Capa 3 (IP) y Capa 4 (TCP/UDP)

#### ICMP Misuse

ICMP no fue diseñado para protegerse de abuso.  
Ejemplos:

- Ping flood
    
- Smurf attack (spoof IP + broadcast ICMP)
    
- Ping sweep para reconocimiento
    

#### TCP SYN Flood

Ataque clásico DoS donde se envían múltiples SYN falsos.  
El servidor mantiene recursos abiertos esperando un ACK que nunca llega.

Es obligatorio conocerlo.

#### UDP Flood

Envía gran cantidad de datagramas UDP a puertos aleatorios para saturar servicios.

---

### 3. Ataques a protocolos

#### DNS Hijacking / DNS Spoofing

Manipular respuestas DNS para redirigir al usuario a sitios falsos.

Ejemplo típico del examen:  
Víctima intenta acceder a “banco.com”, pero recibirá IP de un servidor malicioso.

#### DNS Amplification Attack

Utiliza servidores DNS abiertos para amplificar tráfico y realizar DDoS.

#### DHCP Spoofing

El atacante actúa como servidor DHCP falso.  
Puede asignar:

- Gateway malicioso
    
- DNS malicioso
    
- IPs incorrectas
    

Resultado: tráfico desviado hacia el atacante.

Mitigación: DHCP Snooping (concepto aparece).

#### HTTP Ataques Comunes

- Inyección
    
- Redirecciones falsas
    
- Web spoofing
    
- Session hijacking (secuestrar cookies)
    

#### HTTPS

Preguntan algo importante:  
HTTPS protege contra MITM ya que cifra el tráfico y valida certificados.

---

### 4. DoS y DDoS

Cisco exige la diferencia:

- **DoS**: un solo origen contra un objetivo.
    
- **DDoS**: múltiples orígenes contra un objetivo.
    

Ejemplos:

- SYN flood
    
- Ping flood
    
- UDP flood
    
- HTTP GET flood
    
- Tráfico amplificado (DNS, NTP, CLDAP)
    

---

### 5. Sniffing, Spoofing, Hijacking (diferencias clave)

#### Sniffing

Capturar tráfico.  
Si la red no está segmentada, es más fácil.

#### Spoofing

Falsificar identidad.  
Ejemplo: IP spoofing, ARP spoofing.

#### Hijacking

Tomar control de una sesión existente.  
Ejemplo: robar cookies HTTP para secuestrar sesión.

---

### 6. Ataques físicos (poco, pero lo pregunta)

- Conectar un dispositivo no autorizado a la LAN.
    
- Capturar tráfico desde un puerto expuesto.
    
- Robo de dispositivos.
    

Mitigación:  
Puertos cerrados, Port Security, autenticación a nivel de switch.

## 3. Preguntas tipo examen CCST

1. **¿Qué ataque manipula ARP para redirigir tráfico hacia un atacante?**  
    ARP spoofing.
    
2. **¿Qué ataque llena la tabla CAM del switch para forzarlo a comportarse como un hub?**  
    MAC flooding.
    
3. **¿Cuál es el propósito de un ataque DNS spoofing?**  
    Redirigir al usuario a un sitio malicioso.
    
4. **¿Qué ataque involucra múltiples sistemas enviando tráfico simultáneo hacia un objetivo?**  
    DDoS.
    
5. **¿Qué protocolo puede ser abusado para ataques de amplificación?**  
    DNS.
    
6. **¿Qué ataque ocurre cuando un atacante envía SYN continuos sin completar el handshake?**  
    TCP SYN flood.
    
7. **Un atacante actúa como servidor DHCP falso. ¿Qué ataque es?**  
    DHCP spoofing.
    
8. **¿Cuál es la diferencia entre sniffing y spoofing?**  
    Sniffing captura tráfico; spoofing falsifica identidad.
    

---

## 4. Resumen estratégico para memorizar

- ARP spoofing → MITM → muy preguntado.
    
- CAM flooding → Port Security como mitigación.
    
- DNS spoofing → redirección maliciosa.
    
- DHCP spoofing → gateway malicioso → DHCP Snooping solución.
    
- SYN flood → handshake incompleto.
    
- DDoS = muchos orígenes; DoS = uno.
    
- VLAN hopping requiere mala configuración del switch.
    
- Hijacking = toma control de sesión; sniffing = capturar; spoofing = falsificar.
