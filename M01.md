# Amenazas, Vulnerabilidades y Ataques a la Ciberseguridad

(Formato CCST – Explicación técnica, ejemplos y preguntas tipo)

## 1. Qué debes saber en este módulo

Cisco exige que identifiques claramente:

- Qué es una **amenaza**, qué es una **vulnerabilidad** y qué es un **ataque**.
    
- Tipos de amenazas: malware, phishing, ingeniería social, exploits, ataques de red.
    
- Diferenciar entre **riesgo**, **impacto** y **probabilidad**.
    
- Reconocer vectores de ataque típicos: correo electrónico, USB, web, red interna, WiFi.
    
- Comprender qué es un **actor de amenaza**: script kiddie, hacktivista, cibercriminal, APT.
    
- Entender el ciclo de ataque (Kill Chain).
    

Todo esto aparece fijo en el examen CCST.

---

## 2. Explicación técnica (con ejemplos reales del nivel CCST)

### Amenaza

Cualquier evento o acción potencial que pueda causar daño a un sistema.  
Ejemplo típico: un atacante intentando acceder a un router Cisco sin credenciales.

### Vulnerabilidad

Debilidad explotable en un dispositivo, aplicación o persona.  
Ejemplo:

- Un switch con contraseña por defecto.
    
- Un Windows sin parches.
    
- Un empleado que cae en phishing.
    

### Ataque

Acción que explota la vulnerabilidad para materializar la amenaza.  
Ejemplo: usar un exploit SMB contra Windows 10 sin parchear (EternalBlue).

### Relación entre los tres

Amenaza + vulnerabilidad = **riesgo**.  
Si no hay vulnerabilidad, la amenaza se queda en intención.

Cisco insiste en que memorices esta relación.

## Tipos de amenazas y ataques (visión CCST)

### 1. Malware

Software diseñado para causar daño.  
Incluye virus, gusanos, ransomware, troyanos.

	**Virus**  
	Malware que se adjunta a archivos legítimos y necesita que el usuario ejecute algo para propagarse.
	
	**Gusanos (Worms)**  
	Malware que se replica automáticamente a través de la red sin intervención del usuario.
	
	**Ransomware**  
	Malware que cifra archivos o sistemas y exige un pago para recuperarlos.
	
	**Troyanos (Trojans)**  
	Malware que se oculta dentro de un programa aparentemente legítimo para ejecutar acciones maliciosas.

Ejemplo CCST:  
"Un empleado abre un archivo 'invoice.pdf.exe' y el equipo cifra los archivos."  
Ese es ransomware.

### 2. Phishing / Ingeniería Social

Ataques que explotan el factor humano.  
Ejemplo CCST: correo falso de recuperación de contraseña que te lleva a un sitio réplica.

### 3. Ataques basados en red

Cisco exige conocer al menos estos:

- **DDoS**: saturar un servidor o router con tráfico masivo.
    
- **MITM (man-in-the-middle)**: interceptar comunicaciones en una red WiFi abierta.
    
- **Spoofing**: falsificar direcciones IP o MAC.
    
- **Sniffing**: capturar tráfico no cifrado en un switch mediante ARP poisoning.
    

Ejemplo:  
Un atacante envía paquetes ARP falsificados en la LAN para redirigir tráfico hacia su PC.

### 4. Explotación de vulnerabilidades

Aprovechar fallos de software:

- Buffer overflow
    
- Inyecciones
    
- Parches no aplicados
    
- Servicios expuestos
    
- Versiones antiguas

## 3. Actores de amenaza (lo preguntan muchísimo)

- **Script kiddie**: no tiene conocimientos avanzados; usa herramientas de otros.
    
- **Cibercriminal**: motivación económica.
    
- **Insider**: empleado con acceso legítimo.
    
- **Hacktivista**: motivación política/ideológica.
    
- **APT (Amenaza Persistente Avanzada)**: grupos altamente financiados (gobiernos).
    

Ejemplo CCST:  
"¿Qué tipo de actor tiene recursos y permanece oculto durante meses?"  
Respuesta: APT.

---

## 4. Ciclo de ataque (Cyber Kill Chain)

Modelo clásico que Cisco usa como referencia:

1. Reconocimiento
    
2. Armamento
    
3. Entrega
    
4. Explotación
    
5. Instalación
    
6. Comando y Control (C2)
    
7. Acciones sobre el objetivo
    

Ejemplo aplicado:  
Un atacante escanea puertos → envía phishing → ejecuta payload → mantiene backdoor.

---

## 5. Vectores de ataque

Donde empiezan los ataques:

- Email
    
- Web
    
- USB
    
- Red local
    
- WiFi inseguro
    
- Aplicaciones vulnerables
    

Ejemplo CCST:  
"¿Qué vector de ataque utiliza típicamente un archivo adjunto malicioso?"  
Respuesta: Email.

## 6. Preguntas tipo examen (reales del nivel CCST)

1. **¿Cuál es la diferencia entre amenaza y vulnerabilidad?**  
    Amenaza = posibilidad de ataque; vulnerabilidad = debilidad que permite el ataque.
    
2. **Un atacante envía correos falsos para obtener credenciales. ¿Qué tipo de ataque es?**  
    Phishing.
    
3. **Un router con contraseña por defecto representa qué?**  
    Una vulnerabilidad.
    
4. **¿Qué actor normalmente tiene pocos conocimientos técnicos?**  
    Script kiddie.
    
5. **Un ataque DDoS busca...**  
    Saturar un servicio para que no esté disponible.
    
6. **Cuál es el propósito del C2 en un ataque?**  
    Permitir al atacante controlar el equipo comprometido.
    

---

## 7. Resumen estratégico para memorizar

- Amenaza = intención; vulnerabilidad = debilidad; ataque = acción.
    
- CCST pregunta mucho sobre phishing e ingeniería social.
    
- APT = recursos altos, ataques largos, sigilosos.
    
- Sniffing y spoofing aparecen en preguntas de redes básicas.
    
- La Kill Chain se memoriza en orden.
    
- Malware = software malicioso; troyano finge ser algo legítimo; ransomware cifra archivos.
