## 2. Explicación técnica clara y aplicada al examen

### 1. ¿Qué es la computación en la nube?

Es el uso de recursos:

- de manera remota,
    
- bajo demanda,
    
- administrados por un proveedor (AWS, Azure, Google Cloud),
    
- accesibles por Internet.
    

Ventajas:  
Escalabilidad, disponibilidad, reducción de costos.

---

### 2. Modelos de servicio en la nube (muy preguntados)

#### 1. IaaS (Infrastructure as a Service)

El proveedor ofrece:

- Servidores virtuales
    
- Redes
    
- Almacenamiento
    

El cliente administra:

- Sistema operativo
    
- Aplicaciones
    
- Seguridad interna
    

Ejemplo: máquinas virtuales en AWS o Azure.

#### 2. PaaS (Platform as a Service)

El proveedor ofrece:

- Plataforma de ejecución
    
- Bases de datos
    
- Middleware
    
- Contenedores gestionados
    

El cliente aporta:

- Código
    
- Ajustes de aplicación
    

Ejemplo: Google App Engine, Azure App Service.

#### 3. SaaS (Software as a Service)

El proveedor da el servicio completo.

Ejemplo: Gmail, Office 365, Salesforce.

Pregunta típica:  
“¿Qué modelo requiere que el cliente administre el sistema operativo?”  
→ IaaS.

---

### 3. Responsabilidad Compartida (concepto crucial)

En la nube, la seguridad se divide:

#### El proveedor cloud protege:

- Infraestructura física
    
- Red, hardware
    
- Virtualización
    
- Centros de datos
    

#### El cliente protege:

- Datos
    
- Usuarios
    
- Roles y permisos
    
- Configuración del sistema operativo
    
- Acceso a APIs
    
- Aplicaciones
    
- Contenedores (según modelo)
    

Cisco lo pregunta sí o sí.

### 4. Principales riesgos en la nube

#### Mala configuración (misconfiguration)

El error más común y más peligroso:

- Buckets S3 abiertos al público
    
- Bases de datos sin autenticación
    
- Firewalls cloud mal configurados
    
- Roles excesivos en IAM
    

Ejemplo CCST:  
Un bucket expuesto permite descargar datos sin autenticación.

#### Fuga de datos (data leak)

Pérdida de información sensible por:

- Accesos incorrectos
    
- Error del usuario
    
- Configuración pública
    

#### Robo de credenciales

Uso indebido de:

- Claves API
    
- Tokens
    
- Acceso mediante fuerza bruta
    

#### Ataques a entornos multi-tenant

Errores del proveedor pueden afectar a varios clientes (poco común pero posible).

#### Exposición de servicios

Puertos abiertos en máquinas virtuales.

---

### 5. Controles de seguridad en la nube

#### Identidad y Acceso (IAM)

Reglas para definir:

- Usuarios
    
- Roles
    
- Permisos
    
- Políticas
    

Regla de oro: **Least privilege**.

#### MFA obligatorio

Evita ataques con credenciales robadas.

#### Cifrado

- En tránsito: TLS
    
- En reposo: KMS keys (claves gestionadas)
    

#### Firewalls cloud / Security Groups

Controlan qué tráfico entra y sale de máquinas virtuales.

#### Logging y monitoreo

Servicios como:

- AWS CloudTrail
    
- Azure Monitor
    
- Google Cloud Logging
    

Para auditoría y análisis.

#### Backups y alta disponibilidad

Recuperación ante fallos o incidentes.

---

### 6. Ventajas de seguridad de la nube

- Redundancia automática
    
- Actualizaciones gestionadas
    
- Alta disponibilidad
    
- Seguridad física fuerte
    
- Escaneo continuo
    
- Control de acceso avanzado
    

---

### 7. Limitaciones y riesgos que siguen existiendo

- El proveedor **no protege tu configuración**.
    
- El proveedor **no controla tus contraseñas**.
    
- El proveedor **no evita que expongas tus datos accidentalmente**.
    
- El cliente es responsable de la seguridad interna.

## 3. Preguntas tipo examen CCST

1. **¿Qué modelo cloud requiere que el cliente administre el sistema operativo?**  
    IaaS.
    
2. **¿Qué modelo cloud ofrece software completo gestionado por el proveedor?**  
    SaaS.
    
3. **Quién es responsable de proteger los datos en la nube?**  
    El cliente.
    
4. **Qué riesgo común ocurre por configuraciones incorrectas en cloud?**  
    Misconfiguration → exposición accidental.
    
5. **Qué mecanismo mejora la seguridad de acceso a la nube?**  
    MFA.
    
6. **Qué significa responsabilidad compartida?**  
    Proveedor protege infraestructura; cliente protege datos y configuración.
    
7. **Qué control se usa para gestionar permisos y roles en cloud?**  
    IAM.
    
8. **Qué ocurre si un bucket está mal configurado como “público”?**  
    Cualquiera puede acceder y descargar datos.

## 4. Resumen estratégico para memorizar

- IaaS = tú administras SO; PaaS = tú administras app; SaaS = consumes el servicio.
    
- Responsabilidad compartida = proveedor físico; cliente datos y configuración.
    
- El error más común y más peligroso en cloud = misconfiguration.
    
- IAM = base de seguridad en cloud.
    
- MFA obligatorio = protección crítica.
    
- Cifrado en tránsito (TLS) y en reposo (KMS).
    
- Logging y monitoreo siempre activados.
