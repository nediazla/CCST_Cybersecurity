## 2. Explicación técnica aplicada

### 1. Qué es un endpoint

Un endpoint es cualquier dispositivo de usuario:

- PC
    
- Laptop
    
- Móvil
    
- Tablet
    
- Incluso IoT
    

El atacante normalmente entra por el usuario, no por un firewall.  
Por eso este módulo es tan importante.

---

### 2. Antivirus, Antimalware, Antispyware

#### Antivirus

Detecta amenazas clásicas basadas en firmas.

#### Antimalware

Detecta más tipos de amenazas, incluyendo:

- Ransomware
    
- Rootkits
    
- Troyanos
    
- Gusanos
    

#### Antispyware

Diseñado para evitar programas que capturan datos sin permiso.

Punto clave del examen:  
Hoy día los antivirus modernos son _suites antimalware_.

---

### 3. EDR (Endpoint Detection and Response)

Uno de los conceptos más preguntados.

EDR = herramienta avanzada que:

- Monitorea actividad en el endpoint.
    
- Detecta comportamientos sospechosos.
    
- Bloquea procesos maliciosos.
    
- Permite responder (aislar equipo, eliminar procesos, recolectar evidencia).
    

Ejemplo típico:  
Un proceso en Windows empieza a cifrar archivos → EDR lo detiene.

---

### 4. XDR (Extended Detection and Response)

Extiende la capacidad del EDR hacia:

- Redes
    
- Nube
    
- Email
    
- Identidad
    
- Dispositivos
    

XDR combina señales de múltiples fuentes para ver ataques complejos.

Resumen de examen:  
EDR = endpoint.  
XDR = endpoint + más capas de defensa.

---

### 5. Hardening del endpoint

Hardening = reforzar seguridad del dispositivo mediante:

- Actualizar sistema operativo.
    
- Controlar instalación de software.
    
- Restringir privilegios de usuario.
    
- Firewall activo.
    
- Cifrado (BitLocker, FileVault).
    
- Deshabilitar macros.
    
- Controlar puertos USB.
    
- Evitar servicios innecesarios.
    
- Forzar contraseñas fuertes.
    
- Autenticación multifactor (MFA).
    

Cisco quiere que sepas que hardening **reduce la superficie de ataque**.

---

### 6. Control de aplicaciones y dispositivos

#### Application Control

Permite definir qué software puede ejecutarse:

- Lista blanca: solo apps aprobadas
    
- Lista negra: prohibir apps peligrosas
    

Evita ejecución de malware desconocido.

#### Device Control

Permite controlar dispositivos conectados:

- USB
    
- Bluetooth
    
- WiFi
    
- CD/DVD
    

Ejemplo de examen:  
“¿Qué medida evita que empleados conecten USB no autorizados?”  
→ Device Control.

---

### 7. Parches y actualizaciones

Un endpoint sin parches es un objetivo fácil.  
Actualizaciones corrigen vulnerabilidades críticas.

Ejemplo:  
WannaCry explotó una vulnerabilidad de Windows ya parcheada dos meses antes.

Clave:  
Mantener actualizado el sistema y apps evita explotación de vulnerabilidades conocidas.

---

### 8. Amenazas dirigidas a endpoints

- Ransomware
    
- Troyanos ocultos en PDFs
    
- Keyloggers
    
- Spyware en navegadores
    
- Phishing
    
- Scripts maliciosos (PowerShell, Bash)
    
- Macros maliciosas
    

---

### 9. Detección temprana en endpoints

Se busca identificar:

- Comportamientos anómalos.
    
- Procesos que consumen recursos de forma inusual.
    
- Aplicaciones que intentan ejecutarse sin firma.
    
- Cambios inesperados en archivos críticos.
    
- Conexiones inusuales hacia Internet (C2).
    

---

### 10. Respuesta inicial al incidente

Cisco evalúa conocimiento básico:

1. Aislar el endpoint para evitar propagación.
    
2. Conservar evidencia (no borrar logs).
    
3. Escalar a personal especializado.
    
4. Recuperación mediante backups limpios.

## 3. Preguntas tipo examen CCST

1. **¿Qué herramienta detecta actividad maliciosa en un endpoint y permite aislarlo?**  
    EDR.
    
2. **¿Qué diferencia principal hay entre EDR y XDR?**  
    EDR = endpoint; XDR = múltiples capas (cloud, email, red, identidad).
    
3. **Qué técnica evita instalación de software no autorizado?**  
    Application Control.
    
4. **Qué medida evita el uso de USB no autorizados?**  
    Device Control.
    
5. **Por qué son importantes los parches?**  
    Corrigen vulnerabilidades que pueden ser explotadas.
    
6. **Qué tipo de amenaza cifra archivos y pide rescate?**  
    Ransomware.
    
7. **Qué opción mejora la seguridad de inicio de sesión?**  
    MFA (autenticación multifactor).
    
8. **Qué herramienta evita que un usuario ejecute macros maliciosas?**  
    Políticas de deshabilitar macros.
    

---

## 4. Resumen estratégico para memorizar

- Endpoint = el eslabón débil.
    
- Antivirus = firmas; antimalware = más amplio.
    
- EDR detecta y responde; XDR correlaciona múltiples fuentes.
    
- Hardening = reducir superficie de ataque.
    
- Control de aplicaciones = software autorizado.
    
- Control de dispositivos = bloquear USB.
    
- Actualizaciones = esenciales.
    
- Ransomware = cifra archivos.
    
- MFA = clave para proteger accesos.
