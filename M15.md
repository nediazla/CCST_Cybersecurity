## 2. Explicación técnica clara y aplicada

### 1. ¿Qué es un firewall?

Un firewall es un dispositivo o servicio que **controla el tráfico** entre redes basándose en políticas de seguridad.

Controla:

- Qué tráfico entra.
    
- Qué tráfico sale.
    
- Qué tráfico se bloquea.
    
- Qué tráfico se registra.
    

Es la “primera línea defensiva” en una red.

---

### 2. Tipos de firewalls

Cisco distingue varios:

#### 1. Firewall de filtrado de paquetes (stateless)

Procesa paquetes uno por uno sin contexto.  
Decide **solo** por:

- IP origen
    
- IP destino
    
- Puerto
    
- Protocolo
    

Es rápido, pero no detecta ataques complejos.

#### 2. Firewall con estado (stateful)

Mantiene tabla de conexiones activas.  
Comprende el **estado** de la sesión.

Ejemplo:  
Si un cliente inició una conexión TCP, el firewall permite solo las respuestas correctas.

Este es el más común hoy.

#### 3. Firewall de capa de aplicación (Application Firewall)

Inspecciona tráfico de aplicaciones específicas:

- HTTP
    
- DNS
    
- FTP
    
- SMTP
    

Puede analizar contenido, URLs, comandos de aplicación.

Ejemplo:  
Bloquear solo subdirectorios en HTTP.

#### 4. NGFW (Next-Generation Firewall)

Es el más avanzado:

- Stateful
    
- Inspección profunda de paquetes (DPI)
    
- Identificación de aplicaciones
    
- Filtrado basado en usuario (integración con AD)
    
- IPS integrado
    
- Protección contra malware
    
- Control web (URL filtering)
    

Cisco Firepower es un NGFW.

---

### 3. Cómo funcionan las reglas y políticas del firewall

Cada firewall tiene:

- **Rules**: instrucciones individuales (permitir/denegar).
    
- **Policies**: conjuntos organizados de reglas aplicados a interfaces o zonas.
    

El firewall evalúa reglas desde arriba hacia abajo, igual que ACLs.

---

### 4. Qué inspecciona un firewall

Dependiendo del tipo:

- IP/puertos (todos).
    
- Estado de conexiones (stateful).
    
- Contenido de aplicaciones (NGFW).
    
- Firmas de ataques (NGFW + IPS).
    
- URLs (NGFW).
    
- Identidad del usuario (NGFW con AD).
    

---

### 5. Qué NO hace un firewall

- No evita malware ya dentro del sistema.
    
- No protege contra ataques realizados desde una cuenta válida comprometida.
    
- No evita errores humanos.
    
- No inspecciona tráfico cifrado sin “SSL inspection”.
    
- No reemplaza un IPS avanzado.
    

---

### 6. Diferencia entre firewall, ACL e IDS/IPS

Cisco lo pregunta mucho:

- **ACL** = reglas básicas de filtrado.
    
- **Firewall** = inspección inteligente y control avanzado.
    
- **IDS** = detecta actividad sospechosa, no bloquea.
    
- **IPS** = bloquea tráfico malicioso en tiempo real.
    

NGFW normalmente incluye IPS.

---

### 7. Ubicación típica de firewalls

- Perímetro entre LAN e Internet.
    
- Entre redes internas sensibles.
    
- Entre DMZ y red interna.
    
- En acceso remoto (VPN).
    

---

### 8. Filtrado de tráfico

Los firewalls permiten o deniegan tráfico según:

- Dirección
    
- Puerto
    
- Protocolo
    
- Usuario o grupo
    
- Aplicación
    
- Tipo de archivo
    
- Riesgo asociado

## 3. Preguntas tipo examen CCST

1. **¿Qué hace un firewall?**  
    Controla qué tráfico entra, sale o se bloquea según políticas.
    
2. **¿Qué firewall entiende el estado de las conexiones?**  
    Stateful firewall.
    
3. **¿Qué firewall inspecciona tráfico a nivel de aplicación?**  
    Application Firewall o NGFW.
    
4. **¿Qué firewall incluye IPS, filtrado de URL y control de aplicaciones?**  
    NGFW.
    
5. **¿Qué diferencia principal hay entre ACL y firewall?**  
    ACL solo filtra; firewall inspecciona y aplica políticas avanzadas.
    
6. **¿Qué puede hacer un IPS integrado en un firewall?**  
    Detectar y bloquear ataques.
    
7. **¿Qué limitación tiene un firewall común?**  
    No protege contra errores humanos o tráfico interno legítimo malicioso.
    
8. **¿Dónde suele ubicarse un firewall?**  
    Entre red interna e Internet (perímetro).
    

---

## 4. Resumen estratégico para memorizar

- Firewall = control inteligente de tráfico.
    
- Stateless = básico; Stateful = moderno; NGFW = avanzado.
    
- NGFW integra IPS, control web y detección profunda.
    
- Firewalls ≠ ACLs.
    
- Firewalls inspeccionan tráfico, pero no sustituyen EDR ni previenen errores humanos.
    
- Políticas se aplican por zonas o interfaces.
