# Protección de Redes

(Formato CCST – Conceptos, ejemplos aplicados y preguntas tipo)

## 1. Qué debes saber en este módulo

Para CCST, debes dominar:

- Cómo se protege una red mediante segmentación, VLANs y subredes.
    
- Cómo funcionan los **switches**, **routers** y **firewalls** como elementos de seguridad.
    
- Qué es un **IDS/IPS** y cómo detectan o bloquean ataques.
    
- Concepto de **DMZ** y separación de tráfico interno/externo.
    
- Cómo se protege un punto de acceso inalámbrico.
    
- Concepto de **Zero Trust** a nivel de red.
    
- Medidas básicas: contraseñas, ACLs, actualizaciones, deshabilitar servicios innecesarios.
    

Todo esto es 100% preguntado en CCST.

---

## 2. Explicación técnica (visión directa desde el enfoque Cisco)

### 1. Segmentación de red

Es la práctica de dividir la red en partes más pequeñas para limitar el alcance de un ataque.

Ejemplo:  
Crear una VLAN para administración, otra para usuarios y otra para invitados.  
Si un atacante compromete la red de invitados, **no puede** pivotar hacia la VLAN de administración sin romper otro control.

Ventaja:  
Limita la superficie de ataque y reduce movimientos laterales.

### 2. Switches y seguridad de capa 2

Los switches no son dispositivos de seguridad, pero permiten controles críticos:

- **Port Security**: limita cuántas MAC pueden asociarse a un puerto.  
    Ejemplo típico del examen: “bloquear ataques de CAM table overflow”.
    
- **Deshabilitar puertos no usados**: si un puerto no se usa, se apaga.  
    Evita que un atacante entre físicamente a la LAN.
    
- **VLANs**: separan tráfico, reducen broadcast y ayudan al aislamiento lógico.
    

---

### 3. Routers como barrera de seguridad

Los routers permiten segmentación entre redes diferentes y aplican políticas básicas:

- Filtrado mediante ACLs
    
- NAT para ocultar direcciones internas
    
- Redireccionamiento de tráfico solo permitido
    

Ejemplo CCST:  
“¿Qué dispositivo es el responsable de separar una LAN corporativa de la red ISP?”  
Respuesta: router.

---

### 4. Firewalls

Dispositivo clave para proteger redes.  
Tipos relevantes para CCST:

- **Firewall de filtrado básico (stateless)**
    
- **Firewall con estado (stateful)**
    
- **Firewall basado en aplicaciones**
    
- **Firewall basado en políticas / zonas (ZPF)** → aparecerá más adelante en detalle
    

Concepto fundamental:  
Un firewall decide qué tráfico se permite o bloquea según reglas.

Ejemplo:  
Permitir solo tráfico HTTP/HTTPS hacia un servidor web interno; bloquear todo lo demás.

---

### 5. IDS e IPS

Cisco espera que distingas claramente ambos:

- **IDS (Intrusion Detection System)**: Detecta comportamientos sospechosos pero no bloquea.  
    Ejemplo: Snort.
    
- **IPS (Intrusion Prevention System)**: Detecta y bloquea tráfico malicioso en tiempo real.  
    Ejemplo: Cisco Firepower Threat Defense (FTD) en modo IPS.
    

Preguntan muchísimo esta diferencia.

---

### 6. DMZ (Zona desmilitarizada)

Espacio intermedio donde colocas servicios que deben ser accesibles desde Internet:

- Servidores web
    
- Servidores de correo
    
- Servidores DNS externos
    

Idea central:  
Si el atacante compromete el servidor de la DMZ, **no tiene acceso directo a la red interna**.

---

### 7. Protección de red inalámbrica

Cisco quiere que memorices:

- WPA2/WPA3 = protocolos seguros de autenticación.
    
- Deshabilitar WPS.
    
- Cambiar SSID por defecto, pero **no ocultar SSID** (no mejora seguridad).
    
- Usar contraseñas fuertes en WiFi.
    
- Separar red de invitados.
    

Ejemplo Cisco:  
“¿Qué estándar inalámbrico ofrece la mejor protección actual?”  
Respuesta: WPA3.

---

### 8. Actualizaciones, parches y hardening

Todo equipo de red debe:

- Tener firmware actualizado.
    
- Tener servicios innecesarios deshabilitados.
    
- Tener credenciales seguras en cada dispositivo.
    
- Usar SSH en lugar de Telnet.
    

Ejemplo:  
Un atacante intercepta credenciales porque el administrador configuró Telnet.  
Solución: siempre usar SSH.

---

### 9. Zero Trust aplicado a la red

Principio moderno que Cisco incorpora en CCST:

“Nadie es de confianza por defecto.”

Aplicaciones:

- Autenticación continua
    
- Segmentación estricta
    
- “Least privilege” para cada dispositivo y usuario
    
- Verificación del dispositivo antes de permitir acceso

## 3. Ejemplos tipo examen CCST

1. **¿Qué técnica limita que dispositivos no autorizados se conecten a puertos del switch?**  
    Port Security.
    
2. **Un firewall revisa el estado de una conexión antes de permitir paquetes. ¿Qué tipo es?**  
    Stateful firewall.
    
3. **¿Para qué se usa una DMZ?**  
    Aislar servicios públicos del resto de la red interna.
    
4. **¿Qué dispositivo detecta ataques pero no los bloquea?**  
    IDS.
    
5. **¿Qué método inalámbrico proporciona mayor seguridad?**  
    WPA3.
    
6. **¿Qué tecnología oculta direcciones IP internas hacia Internet?**  
    NAT.
    
7. **Qué beneficio ofrece segmentar la red en VLANs?**  
    Reducir movimiento lateral del atacante.
    

---

## 4. Resumen estratégico para memorizar

- Segmentar = aislar. VLANs = aislamiento lógico.
    
- Firewall decide; IDS detecta; IPS bloquea.
    
- DMZ = zona intermedia para servicios públicos.
    
- Port Security evita ataques de Capa 2.
    
- WPA3 es lo más seguro wireless.
    
- Telnet jamás; SSH siempre.
    
- Zero Trust = nunca confíes por defecto.
    
- Parches y firmware actualizado = requisito básico.
