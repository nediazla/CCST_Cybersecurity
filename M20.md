## 2. Explicación técnica clara y aplicada

### 1. Logs (Registros)

Los logs son **registros cronológicos** de actividades en:

- Sistemas operativos
    
- Firewalls
    
- Servidores
    
- Antivirus/EDR
    
- Routers/Switches
    
- Aplicaciones
    

Ejemplos de lo que puede verse en logs:

- Intentos fallidos de autenticación
    
- Conexiones entrantes
    
- Cambios de configuración
    
- Errores del sistema
    
- Descargas de archivos
    
- Actividad sospechosa
    

Pregunta típica:  
“¿Qué archivo registra intentos fallidos de login en Linux?”  
→ `/var/log/auth.log`.

---

### 2. Eventos vs alertas

Concepto fundamental:

#### Evento

Información registrada sin acción automática.  
Ejemplo:  
“Usuario inició sesión.”

#### Alerta

Notificación generada cuando ocurre algo anómalo o potencialmente malicioso.  
Ejemplo:  
“10 intentos fallidos de autenticación en 5 minutos.”

Cisco quiere que distingas que **alerta = evento que requiere atención**.

---

### 3. SIEM (Security Information and Event Management)

El SIEM es un sistema que:

- Recolecta logs de múltiples fuentes.
    
- Correlaciona eventos.
    
- Detecta patrones sospechosos.
    
- Genera alertas.
    
- Permite investigación.
    
- Provee dashboards y auditoría.
    

Herramientas conocidas:

- Splunk
    
- Elastic SIEM
    
- QRadar
    
- Azure Sentinel
    

Preguntan:  
“¿Qué herramienta combina recolección, correlación y análisis de logs?”  
→ SIEM.

---

### 4. NetFlow (y tecnologías similares)

NetFlow es telemetría creada por Cisco para analizar **flujos** de tráfico.

Un flujo contiene:

- IP origen
    
- IP destino
    
- Puerto origen
    
- Puerto destino
    
- Protocolo
    
- Bytes transferidos
    
- Duración
    

NetFlow NO contiene el contenido del paquete.  
Solo metadatos.

Sirve para:

- Detectar anomalías
    
- Ver picos de tráfico
    
- Identificar DDoS
    
- Identificar conexiones inusuales
    

Otros similares:  
IPFIX, sFlow.

---

### 5. Telemetría de red

Incluye:

- Logs
    
- Flujos
    
- Estadísticas de tráfico
    
- Contadores de interfaz
    
- Alertas de hardware
    

Permite ver comportamiento normal y detectar anomalías.

---

### 6. Actividades sospechosas detectadas por datos de red

- Conexiones a IPs inusuales.
    
- Puertos no usados normalmente.
    
- Aumento de tráfico saliente.
    
- Intentos de escaneo o enumeración.
    
- Repetidos fallos de autenticación.
    
- Tráfico a países donde la empresa no opera.
    
- Patrones de beaconing (C2).
    
- Conexiones a intervalos regulares (malware).
    

---

### 7. Importancia del tiempo y NTP

Todos los dispositivos deben usar la misma hora.

¿Por qué?

- Correlación de eventos en SIEM
    
- Análisis forense
    
- Auditorías
    
- Validación de certificados
    

NTP (Network Time Protocol) mantiene los relojes sincronizados.

---

### 8. Uso de datos de seguridad en investigación

El analista debe:

- Revisar logs relevantes.
    
- Buscar patrones.
    
- Correlacionar flujos NetFlow.
    
- Ver alertas asociadas.
    
- Determinar si es falso positivo o ataque real.

## 3. Preguntas tipo examen CCST

1. **¿Qué es un SIEM?**  
    Sistema que recolecta, correlaciona y analiza logs para generar alertas.
    
2. **¿Qué es NetFlow?**  
    Telemetría de red que muestra flujos: IP origen/destino, puertos, bytes…
    
3. **¿Qué diferencia hay entre un evento y una alerta?**  
    Evento = registro; Alerta = requiere atención.
    
4. **Qué indica un gran aumento de tráfico saliente repentino?**  
    Posible exfiltración de datos o malware.
    
5. **Por qué es importante sincronizar relojes con NTP?**  
    Permite correlación precisa de logs.
    
6. **Qué tipo de datos permite detectar DDoS?**  
    NetFlow (gran volumen de tráfico desde múltiples IPs).
    
7. **Qué herramienta correlaciona eventos y genera alertas?**  
    SIEM.
    
8. **Qué puedes ver en NetFlow?**  
    Puertos, IPs, protocolo, bytes; nunca contenido.
    

---

## 4. Resumen estratégico para memorizar

- Logs = registro histórico.
    
- Evento ≠ alerta.
    
- SIEM = correlación + análisis + alertas.
    
- NetFlow = flujos, no contenido.
    
- Anomalías en tráfico indican posibles ataques.
    
- NTP = sincronización para investigación.
    
- Telemetría sirve para detectar patrones sospechosos.
