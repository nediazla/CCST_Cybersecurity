## 2. Explicación técnica clara y aplicada a CCST

### 1. Qué es una ACL

Una ACL (Access Control List) es una lista de reglas que permiten o deniegan tráfico basándose en:

- Dirección IP origen
    
- Dirección IP destino
    
- Protocolo (TCP, UDP, ICMP…)
    
- Puertos (80, 443, 22…)
    
- Flags
    
- Otros criterios avanzados (no exigidos en CCST)
    

Su propósito principal es **controlar qué tráfico está autorizado**.

---

### 2. ACL estándar vs ACL extendida

#### ACL estándar

Filtra **solo por IP origen**.  
No mira puertos ni destino.  
Se aplica normalmente **cerca del destino**.

Ejemplo:  
“Permitir solo a la IP 192.168.1.50 acceder a la red interna.”

#### ACL extendida

Filtra por:

- IP origen
    
- IP destino
    
- Protocolo
    
- Puerto
    

Es mucho más granular.  
Se aplica normalmente **cerca del origen**.

Ejemplo:  
“Permitir HTTP desde la red 10.0.0.0/24 hacia el servidor 192.168.1.10.”

---

### 3. Orden de evaluación

Las ACL funcionan **de arriba hacia abajo**:

1. La primera regla que coincide con el tráfico **decide**.
    
2. Las reglas siguientes ya no se evalúan.
    
3. Si ninguna coincide → **deny any** implícito.
    

Esto es absolutamente clave.

Ejemplo:  
Si tienes:

1. deny 192.168.1.5
    
2. permit 192.168.0.0/16
    

El host 192.168.1.5 será bloqueado aunque después haya un permit general.

---

### 4. Implicit Deny (negación implícita)

Al final de toda ACL hay una regla invisible:

`deny ip any any`

Esto significa:

- Si el tráfico no coincide con ninguna regla explícita, se deniega.
    
- Es una de las preguntas más clásicas del examen.
    

---

### 5. Inbound vs Outbound

#### Inbound ACL

Filtra tráfico **al entrar** en la interfaz.

Ejemplo:  
Evitar que ciertas IP lleguen al router desde Internet.

#### Outbound ACL

Filtra tráfico **al salir** de la interfaz.

Ejemplo:  
Evitar que usuarios internos accedan a sitios prohibidos.

---

### 6. Aplicaciones típicas en seguridad

- Restringir acceso a redes administrativas.
    
- Permitir solo tráfico HTTP/HTTPS a un servidor web.
    
- Bloquear subredes específicas.
    
- Limitar acceso SSH al router.
    
- Implementar políticas de segmentación.
    
- Prevenir tráfico malicioso entrante.
    

---

### 7. Limitaciones de ACLs

- No detectan malware.
    
- No inspeccionan contenido (capa 7).
    
- No tienen inteligencia contextual.
    
- No reemplazan firewalls avanzados.
    

Aclara Cisco:  
ACLs = control básico.  
Firewalls = inspección profunda y protección avanzada.

## 3. Ejemplos de funcionamiento (explicación conceptual)

### Ejemplo 1: Bloquear una IP concreta

Regla:

`deny 10.10.10.50`

Resultado:  
Esa IP no puede pasar por la interfaz.

### Ejemplo 2: Permitir solo tráfico web hacia un servidor

`permit tcp any 192.168.1.10 eq 80 permit tcp any 192.168.1.10 eq 443 deny   ip any any   ← implícito`

### Ejemplo 3: Permitir tráfico interno y bloquear externos

`permit ip 192.168.0.0 0.0.255.255 any deny   ip any any`

## 4. Preguntas tipo examen CCST

1. **¿Qué es una ACL?**  
    Una lista de reglas que permiten o deniegan tráfico basado en criterios como IP y puertos.
    
2. **¿Cuál es la diferencia entre ACL estándar y extendida?**  
    Estándar: solo IP origen.  
    Extendida: origen, destino, protocolo, puertos.
    
3. **En qué orden se evalúan las reglas de una ACL?**  
    De arriba hacia abajo, la primera coincidencia gana.
    
4. **Qué significa “implicit deny”?**  
    Tráfico no coincidente se bloquea automáticamente al final de la ACL.
    
5. **Qué tipo de ACL se debe colocar cerca del origen?**  
    ACL extendida.
    
6. **Qué tipo de ACL controla solo IP origen?**  
    ACL estándar.
    
7. **Qué ocurre si ninguna regla coincide?**  
    Se aplica el deny implícito.
    
8. **Qué puede hacer una ACL?**  
    Permitir/bloquear tráfico. No analiza contenido.
    

---

## 5. Resumen estratégico para memorizar

- ACL = control de tráfico.
    
- Estándar = origen; Extendida = origen + destino + puertos.
    
- Regla de oro: la primera coincidencia gana.
    
- deny any any implícito.
    
- Inbound = entra a la interfaz; Outbound = sale.
    
- ACLs ≠ firewalls avanzados.
    
- Colocar ACL estándar cerca del destino, extendida cerca del origen.
