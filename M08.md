## 2. Explicación técnica clara y aplicada a CCST

### 1. ¿Qué es Linux?

Linux es un sistema operativo tipo Unix, de código abierto, usado en:

- Servidores web.
    
- Firewalls.
    
- Dispositivos IoT.
    
- Routers y switches (incluyendo Cisco).
    
- Infraestructura cloud.
    
- Sistemas de seguridad.
    

Cisco destaca esto:  
Linux domina gran parte de Internet y la ciberseguridad.

---

### 2. Distribuciones comunes

Todas usan el kernel Linux, pero cambian las herramientas:

- **Ubuntu / Debian**
    
- **CentOS / RHEL**
    
- **Fedora**
    
- **Kali Linux** (orientada a pentesting)
    

En el examen, basta con reconocer que las distros cambian herramientas, pero el sistema base es el mismo.

---

### 3. Uso básico del entorno Linux

#### Terminal

La mayoría de administraciones se hacen por CLI.

Comandos típicos:

- Navegación: `ls`, `cd`, `pwd`
    
- Ver archivos: `cat`, `less`, `tail`
    
- Permisos: `chmod`, `chown`, `ls -l`
    
- Procesos: `ps`, `top`, `kill`
    
- Servicios: `systemctl status`, `systemctl start`
    
- Gestión de red: `ip addr`, `ping`, `ss -tulnp`
    

Solo necesitas comprender su función, no memorizarlos todos.

---

### 4. Usuarios, permisos y privilegios

Modelo de seguridad básico:

- Cada archivo tiene **usuario**, **grupo** y **permisos**.
    
- Permisos: lectura (r), escritura (w) y ejecución (x).
    
- `root` = superusuario.
    

Ataques comunes:

- Ejecutar servicios como root innecesariamente.
    
- Archivos con permisos demasiado abiertos.
    
- Uso indebido de `sudo`.
    

Ejemplo:  
Un script con permisos 777 puede ser modificado por cualquier usuario.

---

### 5. Servicios típicos en Linux

#### SSH

Protocolo de acceso remoto seguro.  
Muy usado y por eso muy atacado:

- Fuerza bruta
    
- Credenciales débiles
    
- Versiones viejas
    

#### Apache/Nginx

Servidores web; vulnerables si no se actualizan.

#### Samba

Servicio de compartición similar a SMB.

#### Cron

Sistema de tareas programadas; usado legítimamente y por malware para persistencia.

---

### 6. Seguridad nativa de Linux

#### Firewall: UFW o iptables / nftables

Permiten abrir/cerrar puertos.

#### Sudo

Permite ejecutar comandos como root.  
Debe usarse con cuidado.

#### SELinux / AppArmor

Capas avanzadas de control de acceso (no muy profundizado en CCST).

#### Actualizaciones

`apt update && apt upgrade` en Ubuntu; fundamental.

Cisco destaca:  
Linux bien configurado es muy seguro; mal configurado es tan vulnerable como cualquier sistema.

---

### 7. Archivos de configuración y logs

Linux guarda configuraciones en texto plano:

- `/etc/passwd`
    
- `/etc/shadow`
    
- `/etc/ssh/sshd_config`
    
- `/etc/hosts`
    
- `/etc/fstab`
    

Logs importantes:

- `/var/log/auth.log`
    
- `/var/log/syslog`
    
- `/var/log/messages`
    

Ejemplo de examen:  
“¿Dónde se registran intentos fallidos de login?”  
→ `/var/log/auth.log`.

---

### 8. Amenazas comunes en Linux

- Ataques de fuerza bruta a SSH.
    
- Permisos incorrectos.
    
- Servicios expuestos.
    
- Malware específico de Linux (aunque menos frecuente).
    
- Scripts maliciosos en cron.
    
- Escaladas de privilegios (kernel exploits).
    
- Exposición de claves SSH privadas.
    

---

### 9. Hardening básico de Linux

- Actualizar siempre.
    
- Deshabilitar SSH root login.
    
- Usar claves SSH en vez de contraseñas.
    
- Firewall estricto.
    
- Remover servicios innecesarios.
    
- Permisos correctos en archivos sensibles.
    
- Revisar `/var/log` regularmente.

## 3. Preguntas tipo CCST

1. **¿Qué es Linux?**  
    Un sistema operativo de código abierto usado extensamente en servidores y dispositivos de red.
    
2. **¿Qué servicio suele ser objeto de ataques de fuerza bruta?**  
    SSH.
    
3. **¿Qué archivo contiene las contraseñas en formato hash?**  
    `/etc/shadow`.
    
4. **¿Cuál es el usuario con privilegios completos sobre el sistema?**  
    root.
    
5. **¿Qué herramienta gestiona firewall en Linux?**  
    iptables/nftables o UFW.
    
6. **¿Qué archivo registra intentos fallidos de inicio de sesión?**  
    `/var/log/auth.log`.
    
7. **Qué herramienta permite ejecutar comandos como root?**  
    sudo.
    
8. **Por qué las tareas cron pueden ser peligrosas?**  
    Pueden usarse como mecanismo de persistencia maliciosa.
    

---

## 4. Resumen estratégico para memorizar

- Linux = código abierto, muy usado en servidores.
    
- root = usuario con control total.
    
- Permisos r/w/x → indispensables.
    
- SSH = acceso remoto, vector común.
    
- `/etc/shadow` → hashes de contraseñas.
    
- `/var/log/auth.log` → intentos fallidos.
    
- firewall = iptables/UFW.
    
- Hardening = actualizar, cerrar puertos, limitar SSH.
    
- Cron = tareas legítimas o persistencia maliciosa.
